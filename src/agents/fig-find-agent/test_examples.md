# Fig Find Agent - GIF Search Test Examples

## 測試案例

以下是用來驗證 fig-find-agent 功能的測試案例：

### 1. 基本情緒表達
```
輸入：開心
預期：回傳開心相關的 GIF
```

### 2. 職場情境
```
輸入：老闆傻眼
預期：回傳適合職場使用的震驚/無語 GIF
```

### 3. 動物主題
```
輸入：貓咪跳舞
預期：回傳貓咪跳舞的 GIF
```

### 4. 慶祝場景
```
輸入：慶祝成功
預期：回傳慶祝/勝利相關的 GIF
```

### 5. 複雜情境描述
```
輸入：尷尬但不失禮貌
預期：回傳適合表達尷尬但禮貌的 GIF
```

### 6. 職場專用請求
```
輸入：我想要一張適合回覆主管的 GIF
預期：回傳職場安全、禮貌的 GIF，並說明適合職場使用
```

### 7. 多結果請求
```
輸入：給我 5 張慶祝成功的 GIF
預期：回傳 5 張不同風格的慶祝 GIF，並分類說明
```

### 8. 商業使用詢問
```
輸入：這張我要商業使用
預期：提醒用戶確認授權和版權問題
```

## 驗收標準

✅ **必須通過的測試**：
- [ ] 輸入「開心」可以回傳開心 GIF
- [ ] 輸入「主管傻眼」可以回傳職場安全的傻眼 GIF
- [ ] 輸入「貓咪跳舞」可以回傳貓咪跳舞 GIF
- [ ] 輸入「給我 5 張慶祝成功 GIF」可以回傳 5 張
- [ ] 輸入「這張我要商業使用」時，會提醒確認授權
- [ ] API Key 缺失時，有明確錯誤提示
- [ ] Agent 基於現有架構修改，保持通用設計

## 錯誤處理測試

### 1. API Key 缺失
```
情境：.env 中沒有設定 TENOR_API_KEY 或 GIPHY_API_KEY
預期回應：
目前 GIF 搜尋 API Key 尚未設定，請先在 `.env` 補上 TENOR_API_KEY 或 GIPHY_API_KEY。
```

### 2. 找不到合適 GIF
```
情境：搜尋非常特殊或不存在的內容
預期回應：
我目前沒有找到很貼切的 GIF。

你可以換一個描述方式，例如：
- 更開心一點
- 更職場一點
- 更可愛一點
- 更誇張一點
```

### 3. 搜尋服務失敗
```
情境：網路問題或 API 服務暫時不可用
預期回應：
GIF 搜尋暫時失敗，請稍後再試，或更換關鍵字。
```

## 使用情境分類測試

### 職場情境關鍵字
- "傳給主管"
- "傳給同事"
- "公司群組"
- "做簡報"

**預期行為**：選擇安全、禮貌、低調的 GIF

### 朋友聊天關鍵字
- "傳給朋友"
- "群組聊天"
- "社群媒體"

**預期行為**：可以更搞笑、誇張的 GIF

## 回應格式測試

### 單一 GIF 回應格式
```markdown
我幫你找到這張比較適合：

![GIF](GIF_URL)

使用情境：
這張適合表達「[具體情境描述]」的感覺。

來源：
[Tenor/GIPHY/網路搜尋結果]

備用連結：
GIF_URL
```

### 多 GIF 回應格式
```markdown
以下是 5 張適合「慶祝成功」的 GIF：

## 1. 熱血慶祝型
![GIF](GIF_URL_1)

## 2. 可愛歡呼型
![GIF](GIF_URL_2)

## 3. 職場可用型
![GIF](GIF_URL_3)

## 4. 搞笑誇張型
![GIF](GIF_URL_4)

## 5. 低調開心型
![GIF](GIF_URL_5)
```
