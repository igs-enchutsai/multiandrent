# 用戶意圖分類指南

## 分類決策樹

```
用戶訊息
├── 包含「多少」「查」「撈」「列出」「有哪些」？
│   ├── 簡單查詢（一句話能回答）→ cha-cha-gin-agent
│   └── 複雜查詢（需要多表 JOIN / 前處理）→ data-hancock
├── 包含「為什麼」「原因」「下降」「上升」「異常」「怎麼了」？
│   └── → monitor-anglo（需先讓 data-hancock 撈資料）
├── 包含「成效」「效果」「因果」「影響」「是否有效」「划算嗎」？
│   └── → analyst-andrew（需先讓 data-hancock 撈資料）
├── 包含「Boss」「魚種」「廳館」「規格」「偏好」「預測」「成功率」？
│   └── → boss-predictor-agent
├── 包含「預測」「趨勢」「未來」「會怎樣」？
│   └── → analyst-andrew
├── 包含「報告」「整理」「摘要」「總結」？
│   └── → 自己整合（可能需要多個 Agent 協作）
└── 其他（閒聊、非業務）？
    └── → 自己回覆
```

## 模糊情境判斷

| 用戶說 | 真正意圖 | 分派 |
|--------|---------|------|
| 「昨天營收怎麼樣」 | 查數據 | data-hancock |
| 「昨天營收怎麼掉了」 | 找原因 | monitor-anglo |
| 「這次 BP 有沒有效」 | 評效果 | analyst-andrew |
| 「幫我看看排行榜」 | 查數據 | data-hancock |
| 「排行榜效果好嗎」 | 評效果 | analyst-andrew |
| 「DAU 最近趨勢」 | 查數據 | data-hancock |
| 「DAU 為什麼一直掉」 | 找原因 | monitor-anglo |

## 追問處理

當用戶追問時：
1. 如果是同一主題的深入 → 繼續派給同一個 Agent
2. 如果是新問題 → 重新分類
3. 如果是「再詳細一點」→ 讓原 Agent 補充
4. 如果是「換個角度看」→ 可能需要換 Agent
